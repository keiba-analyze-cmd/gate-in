## 🎯 思想：「当たったか」でなく「市場に勝ったか」を測る

競馬は◎が1着になる方が少ない。的中率で測ると全員が沈む。
測るのは **「あなたの◎は、控除率込みのオッズ（＝市場の期待値）を上回ったか」**。

- JRA単勝の控除率は約20%。**無策で賭ければ平均 −20%** が基準線。
- レートが上がる条件は「よく当てる」ではなく **「この基準線より良い結果を出す」**。
- → 外れが多いのは**前提**。潰れない。人気より走る馬を見抜ける人だけが伸びる。

## 🧮 ① 1予想のスコア s（of ◎）

```
s = s_core ( + s_partial )          ※ v1 は s_core のみ

s_core = (hit◎ ? ln(O◎) : 0) − b
  O◎   : 確定時の◎馬の単勝オッズ（JRDB SEC pos175）
  hit◎ : ◎馬が1着なら1（同着1着も含む）、それ以外0
  b    : 無策◎の基準スコア（7年データで校正）
```

- **外したら s = −b（必ずマイナス）。** ← ご要望の挙動。
- **外れが多くても潰れない**：b は「無策が横ばいになる高さ」。外れの −b は小さく、人気薄1本の ln(O◎) が複数の外れを取り返す。
- `ln(オッズ)` は人気薄の高配当を圧縮して評価（カザンのモデルと同じ発想）。

**部分点 s_partial（v1.1）**：○/▲が複勝圏 → 加点、危険馬が圏外 → 加点。係数は小さく。v1は◎のみで実装。

## 🧮 ② レート（直近スコアの「収縮平均」）

> ⚠️ 当初案の「ΔR = K×s の加算」は、固定基準に対する加算だと**発散する**（検証で確認）。
> レートは**加算ではなく、収束する平均量**として定義する。

```
状態（DB保存）: m = 直近スコアの指数移動平均, n = 採点済み予想数

1予想ごと:
  n   ← n + 1
  α   = 1 / min(n, Nwin)        // n≤Nwin は累積平均, n>Nwin は地平線Nwinの指数移動平均
  m   ← m + α (s − m)

表示・順位レート:
  収縮  = m × n / (n + C)        // 低サンプルは基準(=0)へ引き戻す
  レート = R0 + scale × 収縮
```

- **有界・安定**：m は平均なので発散しない。無策は0付近、実力者は正、で収束。
- **現在の調子を反映**：Nwin（既定100＝「直近100予想」）で地平線を制御。
- **サンプル数の正直さ**：収縮 C により、低サンプルは基準へ。`n < 30` は **暫定**表示。
- **冪等**：各予想の `s` を保存すれば、履歴からレートを完全再計算できる（校正値変更後の全再計算に必須）。

| `R0` | 初期レート | 1500 | 絶対値は装飾 |

## ✅ 検証結果（合成データ12万予想）

スコア式と収縮平均が狙い通り動くことを確認済み（`rating.test.ts` で再現）。

**収束先（理論レート = R0 + scale×平均スコア）**

| 実力(edge) | レート |
|---|---|
| 0（無策） | **1510**（≈基準で横ばい） |
| 0.10 | 1583 |
| 0.25 | 1704 |
| 0.50 | 1900 |
| 0.80 | 2128 |

→ 無策は基準付近、実力に応じて**単調上昇**、発散なし。

**1予想あたりの変動（確立後）**

| | ΔR |
|---|---|
| 外し | **−8** |
| 的中 人気(O=2.0) | +16 |
| 的中 中穴(O=7.0) | +59 |
| 的中 大穴(O=30) | +109 |

→ 外しは必ず減、的中は人気薄ほど大。大きく上げてもNwinで自然に減衰。

**サンプル数の正直さ**：初手で大穴的中した新人(暫定 ≈1643) < 確立した実力者(≈1782)。

## 🔒 不変条件（検証可能性の生命線）

- **確定後の予想は削除・編集不可**（負け予想を消せると実績の検証可能性が崩壊）。
- 各予想の `score` を保存し、履歴からレートを再計算できる（冪等）。
- ◎馬の取消・除外・オッズ欠損・レース不成立は **無効（更新スキップ）**。

## 🆚 大会ポイントとの棲み分け

**レート＝段位（恒久の梯子）／大会ポイント＝その週のトーナメントスコア。** 通貨は混ぜない。

| | レート | 大会ポイント |
|---|---|---|
| 時間 | 恒久・下がる・ゆっくり | 毎週リセット・加算のみ |
| 対象 | 全予想 | 大会対象レースのみ |
| 数学 | 低分散＝実力 | 高分散でOK＝お祭り（人気薄高配当） |
| 問い | 「あなたは強いか」 | 「今週の優勝は誰か」 |
| 報酬 | 順位・信用・フォロー | ギフト券 |

ポイントの「人気薄で爆発」仕様は**維持**（週次は一発逆転の楽しさが要る）。
大会優勝に小さなレートボーナスは可（ポイントが直接レートになるのは禁止）。
